# Finding 严重级别

## P0

- 会导致数据破坏、权限失守、核心流程不可用，或高概率阻断发布
- 必须有明确触发路径和影响面，不用“可能也许”表述

## P1

- 会导致用户可见错误、核心功能回归、明显错误结果，或常规路径下的环境误判
- 需要说明触发条件、为什么现有实现必然或大概率失败

## P2

- 不是主路径必现，但会在合理边界条件、兼容场景或常见运维操作下出问题
- 需要交代为什么现有测试或实现没有覆盖到它

## P3

- 主要是可维护性、可观测性、可恢复性风险，短期不一定触发用户故障
- 仅在确有实际后果时才写，不把风格偏好当 finding

## 证据要求

- 至少指出：风险点、触发条件、影响、文件位置
- 能引用现有命令输出、测试、配置或调用链时优先引用
- 如果证据不足，只写“待确认问题”而不是升级成 finding
